Introducción¶

En esta tarea se implementar un OCR para reconocer digitos numricos utilizando una red neuronal.

Al igual que en la tarea 1, se trabajar con digitos provenientes de un RUT Chileno.

Esta tarea se implement en Jupyter Notebook con Python, debido a la facilidad para ejecutar cdigo, crear un informe y mostrar los resultados en un mismo documento.

La tarea consiste en entrenar una red neuronal de varias capas (MultiLayer Perceptron) para reconocer dgitos. Este tipo de problemas, de determinar un tipo particular de caso, se le conoce como clasificacin.

Procedimiento¶

El primer paso para este proyecto consiste en entrenar la red. El entrenamiento de la red consiste en entregarle a la red un conjunto de ejemplos de entrada, indicndole a que corresponde cada ejemplo. Adicional a lo anterior se define la cantidad de capas de la red y algunos parmetros tpicos de una red neuronal.

Los ejemplos que se entregarn a la red, en esta tarea se entregarn de dos formas:

  1. Histograma de orientaciones (Tarea 1).
  2. Imagen Binaria Vectorizada y reducida.

El histograma de orientaciones, como se vi en la tarea 1, consiste en calcular, para cada pixel, el ngulo y la magnitud de la orientacin de cambios entre los pixeles vecinos. Se realizaron modificaciones a la tarea 1, esta vez se trabaj con Sobel como fue sugerido anteriormente y con 2 celdas nicamente. Tambin se trabaj con 16 bins en vez de solo 9. Esto di resultados visiblemente mejores en el histograma.

Con respecto al segundo mtodo, el procesamiento es ms sencillo. Se convertir la imagen a 28x28 pixeles y posteriormente se convertir a un arreglo plano, sin ningn procesamiento ms que la binarizacin.

Con ambos mtodos se proceder a entrenar la red neuronal. Una vez entrenada la red, se guardar el modelo y posteriormente se utilizar para clasificar los dgitos de un RUT.

Dado lo anterior, el procedimiento que se utilizar es el siguiente:

  1. Entrenar la red neuronal
    • Procesar las imgenes de entrada segn:
      • Histograma de Orientaciones
      • Vector de Imagen Binarizada
    • Entrenar y validar un modelo con distintos parmetros:
      • Cantidad de neuronas ocultas:
        • 10

      • Funcin de activacin:
        • Sigmoidal (logistic)
        • TanH

  2. Guardar el modelo entrenado usando joblib de sklearn.externals
  3. Clasificar los dgitos de la imagen

def getOrientAndMags(image):
    gradX = filters.sobel_h(image)
    gradY = filters.sobel_v(image)
    o = (np.arctan2(gradY, gradX)  * 180 / np.pi) % 360
    m = np.sqrt(gradY**2 + gradX**2)
    return o, m

def getDigitHistogram(image, bins, assignment = "soft", threshold = 0.1):
    hist = np.zeros(bins, dtype=np.float32)
    w, h = image.shape
    
    o, m = getOrientAndMags(image)
    maxVal = np.max(m) * np.max(o)
    
    for i in range(w):
        for j in range(h):
            # Skip those with magnitude lower than 'threshold'.
            if(m[i,j] > threshold):
                calc = (o[i,j] * m[i,j])
                if(assignment == "hard"):
                    kbin = int(round(calc/maxVal*(bins-1)))
                    hist[kbin] += 1.0
                else:
                    b = calc/maxVal*(bins-1)
                    if(math.isnan(b)):
                        b = 0
                    dist = b - math.floor(b)
                    lbin = math.floor(b)
                    rbin = math.ceil(b)
                    if(dist < 0.5):
                        lw = 0.5 - dist
                        rw = 1-lw
                    else:
                        rw = dist - 0.5
                        lw = 1 - rw
                    hist[lbin] += lw
                    hist[rbin] += rw
    return hist

def fixImagePadding(image, cellSize = (4,4)):
    w, h = image.shape
    cX, cY = cellSize
    padX = cX - (w % cX)
    padY = cY - (h % cY)
    if(padX == cX): padX = 0
    if(padY == cY): padY = 0
    paddedImage = np.zeros((w+padX, h+padY))
    paddedImage.fill(255)
    for x in range(w):
        for y in range(h):
            paddedImage[x,y] = image[x,y]
    return paddedImage

def getImageCells(image, cellSize = (4,4)):
    cells = []
    w, h = image.shape
    cX, cY = cellSize
    for x in range(int(w/cX)):
        for y in range(int(h/cY)):
            cells.append(image[x*cX:((x+1)*cX), y*cY:((y+1)*cY)])

    return cells

def concatenateCellsDescriptors(cells):
    concat = []
    for cell in cells:
        hist = getDigitHistogram(cell, bins = nBins, assignment = assignment, threshold = magnitudeThreshold)
        for h in hist:
            concat.append(h)
    return concat

def resizeAndGetDigitHistogramDescriptor(digitFilename, w = 0, h = 0, cW = 0, cH = 0, verbose = True):
    digit=io.imread(digitFilename)
    digitImage = digit
    if(w>0 and h >0):
        digit = misc.imresize(digit, (w, h))
    if(cW>0 and cH>0):
        digit = fixImagePadding(digit, (cW, cH))
    else:
        cW, cH = digit.shape
    cells = getImageCells(digit, (cW, cH))
    concat = concatenateCellsDescriptors(cells)
    
    if(verbose):
        plt.imshow(digit, cmap="gray")
        print("ImageFilename:", digitFilename)
        print("original:", digitImage.shape, "Resize and padded:", digit.shape)
        print("Concatenated histogram length:", len(concat))
        plotCellDescriptors(concat, cells)
    
    return np.array(concat), cells

def getImageVector(digitFilename, w = 28, h = 28):
    digitImage=io.imread(digitFilename)
    digit = misc.imresize(digitImage, (w, h))
    th=filters.threshold_otsu(digit*0.95)
    digit = digit > th
    return np.reshape(digit, w*h)

Conclusiones y Resultados¶

En este proyecto se han entrenado varias redes neuronales con distintos parmetros y distintos Feature Vectors (Histograma y Vectorizacin Binaria + Reduccin).

Se puede apreciar de los resultados, que trabajar con una red con menos neuronas, conlleva en la mayora de los casos a peores resultados pero tiempos de entrenamientos mayores.

Al entrenar con 10 neuronas, muchas veces el entrenamiento conclua de forma abrupta al llegar a avanzar bajo la tolerancia. Esto sin embargo, como se puede apreciar en los modelos de entrenamiento, es variable ya que como se vi en clases, los pesos iniciales determinan en gran medida el entrenamiento de la red. Al tomarse estos al azar, puede llegar a encontrarse un mnimo global o un mnimo local. Al entrenarse la red con ms neuronas, la probabilidad de encontrar mximos globales es ms alta.

Algunos resultados obtenidos (utilizando el mtodo de los vectores) que apoyan lo anterior fueron:

  • RELU, 10 neuronas, 29%
  • RELU, 10 neuronas, 45%
  • RELU, 10 neuronas, 82%
  • RELU, 10 neuronas, 85%

Los resultados sobre las funciones de activacin no fueron del todo desicivos. Se puede apreciar que varios intentos de entrenamiento con 10 neuronas y distintas funciones de activacin fueron diferentes:

  • Log ~0%-48%,
  • RELU ~30%-85%
  • TANH ~0%-52%

De todas formas, como se mencion anteriormente, estos podran bien ser casualidad ya que se encontr que los resultados fueron bastante similares con 50 neuronas y se podra estimar que los tiempos de entrenamiento tambin son similares. Se considera que se requerira de ejemplos con ms clases y ms dimensiones para determinar correctamente cul de las tres funciones de activacin entrega mejores resultados.

Los resultados obtenidos para los dos mtodos son mejores a los obtenidos en la tarea 1. Esto principalmente a que se tenan ms ejemplos para entrenamiento que en la vez pasada. En la tarea 1 se entren una clasificador con 40 ejemplos, en este caso 6600 ejemplos para pruebas. De lo mismo, sin una experimentacin ms amplia no se podra indicar directamente si para esta aplicacin en especfico, un resultado es mejor que otro.

Un problema que se tuvo fue entrenar con el vector de orientaciones. Los resultados no fueron para nada satisfactorios.

Los mejores resultados que se tuvieron con ste mtodo fueron:

  • Logistic, 50, 61%
  • RELU, 50, 55%
  • TanH, 50, 60%

Lo anterior se supone por varias cosas: Los vectores de entrada eran muchos ms pequeos, la imagen vectorizada era un vector de entrada de 768 valores, mientras que en el caso de las orientaciones se trabaj con 16 bins y 2 celdas, osea, 32 valores de entrada.

Entrenar la red neuronal tom mucho ms tiempo. Si bien esto no afecta en los resultados, finalmente, para poder entregar resultados se trabaj con una reduccin del tamao de las imgenes. Esto en parte debido a que el procesamiento de las imgenes trabajaba con imgenes mucho ms grandes (128x128 pixeles). Se modific el cdigo de la tarea 1 segn los comentarios realizados post-entrega, sin embargo, los resultados sin reducir los tamaos de las imgenes requirieron ms tiempo y de esta manera, no se pudo experimentar tanto como con el otro mtodo.
